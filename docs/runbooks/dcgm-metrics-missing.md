# Runbook: DCGM Metrics Missing

## Purpose
Troubleshoot cases where Prometheus is not receiving NVIDIA GPU metrics from the DCGM exporter.

## Symptoms
- No GPU metrics appear in Prometheus.
- Queries such as `DCGM_FI_DEV_GPU_UTIL` return no data.
- The DCGM exporter pod is running but scrape targets appear unhealthy.

## Step 1: Check exporter pod health
```bash
kubectl -n monitoring get pods -l app.kubernetes.io/name=dcgm-exporter
kubectl -n monitoring logs deploy/dcgm-exporter --tail=200
```

## Step 2: Check ServiceMonitor
```bash
kubectl -n monitoring get servicemonitor | grep -i dcgm
kubectl -n monitoring describe servicemonitor dcgm-exporter
```

## Step 3: Check Prometheus targets
```bash
kubectl port-forward -n monitoring svc/kube-prometheus-stack-prometheus 9090:9090
```

Then in the Prometheus UI:
- Open Status -> Targets.
- Confirm the DCGM exporter target is `UP`.

## Step 4: Check the Prometheus ConfigMap
The exact ConfigMap name is generated by the Helm release, so inspect the available ConfigMaps first:
```bash
kubectl -n monitoring get cm | grep -i prometheus
```

Then inspect the most relevant one:
```bash
kubectl -n monitoring describe cm <prometheus-configmap-name>
```

## Common fixes
- Recheck the DCGM exporter pod and node selector.
- Confirm the GPU node is labeled and tainted correctly.
- Confirm the ServiceMonitor matches the release label used by kube-prometheus-stack.
- Restart the Prometheus pod if the scrape config was updated recently.
